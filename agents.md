# Precog local build / install + autocomplete-popup debugging handoff

This documents a multi-session effort to build the **Precog** CLI fork (an Amazon Q / Fig
descendant) from source on macOS and get the **inline autocomplete popup** working in iTerm2.
Build/install is fully working. The popup pipeline is *almost* working — read "Current status"
and "Leading hypothesis" before doing anything.

Platform: macOS (Darwin 25.3.0, Apple Silicon). Shell: zsh. Terminal: iTerm2.
Workspace version: `1.19.7` across all crates.

---

## 1. What this project is

Hybrid Rust + JS monorepo (cargo workspace + pnpm/turbo). Key pieces relevant to autocomplete:

| Crate / package | Binary / output | Role |
|---|---|---|
| `crates/q_cli` | `precog` | the CLI (`precog`, `precog doctor`, `precog integrations …`) |
| `crates/figterm` | `precogterm` | PTY wrapper that intercepts the shell, sends the "edit buffer" to the desktop app over a socket |
| `crates/fig_desktop` | `precog_desktop` | the desktop app (wry/tao webviews): renders dashboard + autocomplete overlay, does macOS AX/IME caret tracking |
| `crates/fig_input_method` | `fig_input_method` | macOS input method (IMKit), bundled as `CodeWhispererInputMethod.app` |
| `packages/dashboard-app` | `dist/` | dashboard webview |
| `packages/autocomplete-app` | `dist/` | autocomplete overlay webview (the popup UI) |
| `packages/autocomplete-specs` | `build/` | compiled Fig command specs, served over `spec://localhost` |

Binary-name constants live in `crates/fig_util/src/consts.rs`
(`CLI_BINARY_NAME=precog`, `PTY_BINARY_NAME=precogterm`, etc.).

## 3. Install layout (what we manually placed — this is the working layout)

`Precog.app` already existed at `/Applications/Precog.app` (from an older May-7 install).
`Contents/Helpers/` and the `Helpers/precog` binary inside it are **root:wheel** (need sudo).
`Contents/MacOS/`, `Contents/Resources/` are **user-owned** (no sudo needed).

```
/Applications/Precog.app/Contents/
  MacOS/
    precog            -> REAL q_cli binary  (NOT a symlink — see "symlink loop" gotcha)
    precog_desktop    -> REAL fig_desktop binary, ad-hoc codesigned
  Resources/
    dashboard/             <- cp -R packages/dashboard-app/dist
    autocomplete/          <- cp -R packages/autocomplete-app/dist
    autocomplete-specs/build/  <- cp -R packages/autocomplete-specs/build/.  (compiled specs)
  Helpers/
    precog                 (old root-owned CLI copy; unused, leave it)
    CodeWhispererInputMethod.app   <- IME bundle (copied here once via sudo)

~/.local/bin/
  precog               -> symlink to /Applications/Precog.app/Contents/MacOS/precog
  precogterm           -> REAL figterm binary
  bash (precogterm)    -> figterm copy   (per-shell named copies, required by the launch hook)
  zsh (precogterm)     -> figterm copy
  fish (precogterm)    -> figterm copy
  nu (precogterm)      -> figterm copy

~/Library/Input Methods/
  CodeWhispererInputMethod.app -> symlink into /Applications/Precog.app/Contents/Helpers/...
```

IME bundle is assembled in-repo at `build/CodeWhispererInputMethod.app` then copied to Helpers/:
```
build/CodeWhispererInputMethod.app/Contents/
  MacOS/fig_input_method      <- target/release/fig_input_method
  Info.plist                  <- crates/fig_input_method/Info.plist (generated by its build.rs)
  Resources/                  <- crates/fig_input_method/resources/*
```
Then: `codesign --force --deep --sign - build/CodeWhispererInputMethod.app` and
`sudo cp -R build/CodeWhispererInputMethod.app /Applications/Precog.app/Contents/Helpers/`.

### Reinstall (after a rebuild) — use the scripts

Reinstall is now scripted with a **stable signing identity** so the Accessibility
(TCC) grant survives rebuilds (see §8 #4 — this was the recurring popup-killer):

```sh
ops-scripts/create-signing-cert.sh        # ONE TIME: makes the "Precog Dev" identity
ops-scripts/reinstall.sh --reset-tcc      # first reinstall: re-grants Accessibility once
ops-scripts/reinstall.sh                  # every rebuild after that
```

After the first `--reset-tcc` run: System Settings → Privacy & Security →
Accessibility → enable **Precog**, then fully quit + reopen iTerm.

## 5. The autocomplete popup pipeline (how it's supposed to work)

1. `precogterm` wraps the shell, streams the typed "edit buffer" to `precog_desktop` via a unix socket.
2. `precog_desktop` receives `EditBufferChanged`, forwards it to the autocomplete webview
   (`Contents/Resources/autocomplete/`), which matches input against specs served over
   `spec://localhost` from `Contents/Resources/autocomplete-specs/build/`.
3. `precog_desktop` computes the **caret screen position** and positions the overlay window there.
   - Position source depends on terminal: `crates/fig_util/src/terminal.rs`
     - `supports_macos_input_method()` → IME path (Alacritty, Kitty, WezTerm, Ghostty, …)
     - `supports_macos_accessibility()` → **AX path (iTerm2, Terminal.app, VSCode, Hyper, …)**
   - For **iTerm2 the AX path is used** (`crates/fig_desktop/src/platform/macos.rs:673` `refresh_window_position`,
     calls `get_cursor_position()` → `macos_utils::caret_position::get_caret_position`).
   - AX reads `AXSelectedTextRange` + `AXBoundsForRange` on the focused element. If the rect is
     `width==0 && height==0` it returns INVALID and logs
     `"Prevents flashing autocomplete in bottom corner"` (`caret_position.rs:138`).
4. The autocomplete webview requests its own window size via `positionWindowRequest`. Empty/no
   suggestions → it asks for ~`1x1`..`36x16` (effectively invisible). Real suggestions → a large size.

## 6. Debugging journey & what was fixed (chronological)

1. **Q_TERM version mismatch** in `precog doctor` — the running term wrapper was the *Kiro CLI*'s
   figterm. Fixed by building/installing `precogterm` (per-shell copies) so the precog shell hook
   exec's our wrapper. (`precog init zsh pre` launches `precogterm` if found on PATH.)
2. **`precog` opened the dashboard but showed "not found"** — `Contents/Resources/dashboard` and
   `…/autocomplete` were missing from the bundle. Fixed by copying the built `dist/`s in.
3. **Input-method install failed** (`Could not load bundle …/Helpers/CodeWhispererInputMethod.app`)
   — bundle didn't exist. Built `fig_input_method`, assembled the `.app`, sudo-copied to Helpers/.
4. **`precog integrations install input-method` → "Could not run precog cli: No such file"** — the
   installer shells out to `<app>/Contents/MacOS/precog` (see
   `crates/fig_integrations/src/input_method/mod.rs:483`), which didn't exist. Fixed via symlink/real
   binary at `Contents/MacOS/precog`. Then it printed **"Installed!"**.
5. **Popup flashed then vanished** — desktop log showed
   `WARN macos_utils::window_server:374 Error setting up tracking for 'com.googlecode.iterm2'`.
   Caused by a **stale Accessibility (TCC) grant** after we re-codesigned the desktop binary
   (ad-hoc signing changes identity → TCC entry no longer matches). Fixed with
   `tccutil reset Accessibility dev.precog.cli` + re-grant + **full iTerm restart**. After that the
   AX caret reads became valid again (`CaretPosition { valid: true, … }`).
6. **Still nothing after caret fixed** — desktop log then showed the full happy path:
   `Sending edit buffer change notification to webview` + `Sending caret update` +
   `Position Window Request … window_id=autocomplete`, but window sizes were only `1x1`/`6x6`/`36x16`.
   Root cause: `Contents/Resources/autocomplete-specs/build/` contained **only `icons/`** — the
   compiled specs (`index.json` ~21K, `dynamic/`, `index.js`) were missing. Fixed by
   `cp -R packages/autocomplete-specs/build/. <bundle>/…/autocomplete-specs/build/`.
7. **Symlink loop** (`zsh: too many levels of symbolic links: precog`) — at one point BOTH
   `~/.local/bin/precog` and `Contents/MacOS/precog` were symlinks pointing at each other (the
   desktop app re-creates `~/.local/bin/precog -> Contents/MacOS/precog` on launch). Fixed by making
   `Contents/MacOS/precog` the **real** binary and `~/.local/bin/precog` the symlink to it.
8. Did a full clean rebuild + reinstall of all 4 binaries + JS apps + specs. All report `1.19.7`.

## 8. Leading hypothesis on the remaining error (popup renders empty / no suggestions)

The pipeline is healthy; the open question is purely **"why does the autocomplete webview render no
suggestions (window stays ~1x1..36x16)?"** Most likely, in order:

1. **Specs weren't being served** (most probable, and the last thing fixed) — the bundle's
   `autocomplete-specs/build/` had only `icons/`. Now populated with `index.json`/`dynamic/`/`index.js`.
   If suggestions still don't appear after the restart, verify the `spec://` protocol actually
   resolves to that dir: `crates/fig_desktop/src/protocol/spec.rs` (search order includes
   `<exe-dir>/../Resources/autocomplete-specs/build`). Add logging there or check for `spec://`
   404s. NOTE the search path is **relative to the exe dir**; because `Contents/MacOS/precog_desktop`
   is real (not a symlink) `exe_dir/../Resources/...` resolves correctly — but double-check macOS
   isn't running it through a translocated/`AppTranslocation` path.

2. **Webview JS errors** — the definitive check is the autocomplete webview's devtools console:
   `precog debug devtools` (subcommand under `precog debug`; pick the autocomplete webview). Look for
   failed `spec://` / `api://` fetches, or JS exceptions in the `@amzn/autocomplete` engine. The
   autocomplete-app is built from `packages/autocomplete-app` and bundles the `@amzn/autocomplete`
   engine; a build/version mismatch there would render an empty list.

3. **Accessibility re-break after the last resign** — the final reinstall **re-codesigned
   `precog_desktop` ad-hoc again**, which can re-invalidate the TCC grant (same as step 5 above). If
   the popup is back to *flash-then-vanish* (rather than empty), it's the caret/AX path again:
   re-grant Accessibility for `dev.precog.cli` and restart iTerm. Confirm via `fig_desktop.log`
   whether you see `valid: true` carets vs `Prevents flashing autocomplete in bottom corner`.

4. **Unstable ad-hoc signature is the systemic root cause of the AX flakiness.** Every
   `codesign --force --sign -` produces a new identity (the app's designated requirement is then a
   bare `cdhash`, which changes every build), so the Accessibility grant breaks on every rebuild →
   AX caret read fails with `kAXErrorCannotComplete (-25204)` → `caret_position.rs:138` logs
   "Prevents flashing autocomplete in bottom corner" → popup never appears.
   **FIXED (2026-05-27) — now scripted:** `ops-scripts/create-signing-cert.sh` creates a stable
   self-signed "Precog Dev" code-signing identity once; `ops-scripts/reinstall.sh` signs the
   binaries we own with it (NOT `--deep`, because root-owned `Contents/Helpers/precog` would fail —
   `fig_desktop` is the AX-requesting process, so signing it is what the TCC grant tracks). The
   designated requirement becomes `identifier "dev.precog.cli" and certificate leaf = H"…"`, which is
   constant across rebuilds, so you grant Accessibility once (`reinstall.sh --reset-tcc` on first run)
   and stop re-fighting TCC. Verify with `codesign -d -r- …/precog_desktop` — it should reference
   `certificate leaf`, not a bare `cdhash`.

5. **Invalid BUNDLE seal silently denies Accessibility (`-25211`).** `Precog.app` is a *sealed*
   bundle (`Contents/_CodeSignature/CodeResources` hashes every nested file). Signing/replacing the
   inner binaries (`MacOS/precog`, `MacOS/precog_desktop`) **without re-sealing the bundle** leaves
   the bundle signature invalid — `codesign --verify --strict /Applications/Precog.app` reports
   `nested code is modified or invalid` (`file modified: …/MacOS/precog`). macOS then refuses to honor
   the Accessibility grant and `caret_position.rs:59` logs a tight loop of
   `accessibility error -25211` (`kAXErrorAPIDisabled`) — toggling the permission on does nothing.
   **Fix:** sign the inner binaries, then sign the **whole bundle LAST** (NOT `--deep`, so root-owned
   `Helpers/*` are sealed by reference, not re-signed):
   `codesign --force --sign "Precog Dev" /Applications/Precog.app`, then confirm
   `codesign --verify --strict /Applications/Precog.app` → `valid on disk / satisfies its Designated
   Requirement`. `ops-scripts/reinstall.sh` now does this and aborts if validation fails. After a
   re-sign the grant is stale, so `--reset-tcc` + re-grant + iTerm restart once.
